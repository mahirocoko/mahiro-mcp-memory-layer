# Task 1 Console Contract Characterization

## Current Reality

- Page routes are exactly `/`, `/review`, `/rejected`, and `/graph` in `src/features/memory-console/server.ts`.
- Action routes are exactly `/actions/review`, `/actions/promote`, and `/actions/purge-rejected` in `src/features/memory-console/server.ts`.
- Page routes accept `GET` and `HEAD`. The server renders the same page result for both methods, then suppresses the response body for `HEAD` before ending the HTTP response.
- `/` loads filtered canonical memories through `loadConsoleMemories()` and renders `renderMemoryConsolePage()`.
- `/review` forces inbox-oriented filters (`view=inbox`, `verificationStatus=hypothesis`, `reviewStatus=pending`), loads review queue data through `loadConsoleReview()`, and renders `renderReviewConsolePage()`.
- `/rejected` forces rejected-quarantine filters (`view=firehose`, `verificationStatus=all`, `reviewStatus=rejected`), loads filtered canonical memories through `loadConsoleMemories()`, and renders `renderRejectedConsolePage()` through `renderRoutePage()`.
- `/graph` forces project-oriented filters (`view=projects`), loads graph data through `loadConsoleGraph()`, and renders `renderGraphConsolePage()`.
- `/actions/review` accepts only `POST`, validates review form input, runs `reviewMemory`, and returns `303` with `Location` defaulting to `/review` unless the form supplies the supported redirect.
- `/actions/promote` accepts only `POST`, validates promote form input, runs `promoteMemory`, and returns `303` with `Location: /review`.
- `/actions/purge-rejected` accepts only `POST`, validates purge form input, runs `purgeRejectedMemories`, and returns `200 text/html` with the purge result page instead of redirecting.
- Page loads are read/list/search oriented. `createLocalMemoryConsoleBackend()` wires local `JsonlLogStore`, `LanceDB`, `RetrievalTraceStore`, and `MemoryService`, while read paths expose `readAll`, `list`, and `search` for console page loading.
- Mutation-capable behavior stays behind action writers (`reviewMemory`, `promoteMemory`, `purgeRejectedMemories`) and is exercised only by POST action routes.
