## Lessons Learned

- `contextSize` is trace/context metadata, not a hit/non-hit indicator; use returned IDs and `degraded` to explain retrieval state.
- Empty success needs next actions, not just a label: check `projectId`, `containerId`, scoped durable records, and memory seeding/reset state.
- Plan compliance evidence should distinguish implementation files from orchestration bookkeeping when `.sisyphus` state changes during work.
- Docs regression tests should protect exact contract strings while avoiding full-document snapshots.
- Final review rejection is useful when it names fixable gaps; rerun only the rejected reviewer after targeted remediation.
