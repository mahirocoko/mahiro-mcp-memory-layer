## Lessons Learned

- Cross-module invariants need tests that compare produced artifacts against manifest metadata, not only unit tests per module.
- Explicit output override paths are dangerous when the writer performs whole-directory replacement; safety should reject broad repo paths before any staging or rename work starts.
- Final-wave evidence should be treated as the canonical acceptance signal, especially when regular implementation tasks have already passed.
- Retrospective metrics are local derived snapshots; mention them as context without pretending they are complete telemetry.
