## Lessons Learned

- A passing UI build and browser smoke test does not prove destructive JSON action validation is fail-closed.
- For guarded purge APIs, validation must reject malformed arrays wholesale instead of filtering to a partial valid subset.
- Final-wave reviewer rejections should be treated as useful signal, not process noise; rerun only the rejecting reviewer after a targeted fix.
- When tool output truncates reviewer summaries, session search for `VERDICT`, `APPROVE`, and `REJECT` is a reliable recovery path.
