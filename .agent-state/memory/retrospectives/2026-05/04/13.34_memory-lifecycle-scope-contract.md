## Lessons Learned

- Exact-scope filters need explicit diagnostics for sibling scopes; otherwise valid memory looks absent.
- Stable project names are safer recall identities than volatile host project IDs when both are available.
- MemPalace concepts are useful as principles, but this repo should avoid adopting palace vocabulary as public API.
- PreCompact is the most valuable lifecycle checkpoint to harden next, but it must share idempotency with idle persistence.
- Hook compatibility belongs outside the memory package; memory should consume lifecycle signals, not execute hooks.
